# 🐍 30 Days of Python

Hey there! 👋

Welcome to my **30 Days of Python** journey. If you've ever wanted to learn Python but felt overwhelmed by where to start, you're in the right place. This repo breaks Python down into 30 small, digestible days — one topic at a time, no fluff, no pressure.

I put this together while learning Python myself, so trust me — everything here is written with a beginner's struggles in mind. If I got stuck on something, I made sure to explain it in a way that actually makes sense.

## 📖 What is this course?

It's exactly what it sounds like: 30 days, 30 topics, from "Hello World" all the way to building your own API. Each day has its own folder with notes, examples, and exercises you can actually run and play with.

Here's the journey:

| Days | What you'll learn |
|------|-------------------|
| 1–5 | The basics — introduction, variables, operators, strings, and lists |
| 6–10 | Core data structures — tuples, sets, dictionaries, conditionals, and loops |
| 11–15 | Leveling up — functions, modules, list comprehension, higher order functions, and type errors |
| 16–20 | Real-world skills — date & time, exception handling, regex, file handling, and pip |
| 21–25 | Getting serious — classes & objects, web scraping, virtual environments, statistics, and pandas |
| 26–30 | The big leagues — Python for web, MongoDB, working with APIs, building your own API, and wrapping it all up |

## 💡 Why does this matter?

Honestly? Because Python is everywhere.

- **It's beginner-friendly.** Python reads almost like plain English. You spend less time fighting syntax and more time actually building things.
- **It opens doors.** Data science, web development, automation, AI, machine learning — Python is the common thread through all of them.
- **Consistency beats intensity.** That's the whole philosophy of this course. You don't need to study 8 hours a day. Just show up, do one day at a time, and in a month you'll be surprised how far you've come.
- **Learning by doing works.** Every day has exercises. Don't just read the code — type it out, break it, fix it. That's where the real learning happens.

## 🚀 How to get started

1. Make sure you have [Python](https://www.python.org/downloads/) installed (version 3.8 or higher is fine).
2. Clone this repo:
   ```bash
   git clone https://github.com/djsmacker01/30-Days-Of-Python.git
   ```
3. Start with [Day 1](./30-Days-Of-Python-master/01_Day_Introduction) and go at your own pace.
4. Miss a day? No guilt. Just pick up where you left off. The "30 days" is a guide, not a deadline. 😊

## 🤝 This is open source — come build it with me!

Here's the thing: I don't want this to be *my* course. I want it to be *ours*.

If you spot a typo, have a better way of explaining something, want to add more exercises, or translate content into another language — **please contribute!** No contribution is too small. Seriously, even fixing a single spelling mistake helps the next learner.

**How to contribute:**

1. Fork this repo
2. Create a branch for your change (`git checkout -b my-improvement`)
3. Make your changes
4. Open a pull request and tell me what you improved

If you're new to open source, this is a great place to make your first contribution. I'll do my best to review PRs and help you along. Everyone starts somewhere. 💚

**A few gentle ground rules:**

- Keep it simple. This course is for beginners — if a 12-year-old can't follow the explanation, let's simplify it.
- Be kind in discussions and reviews. We're all learning here.
- Test your code examples before submitting. Broken examples are worse than no examples!
